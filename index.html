Breadth-First Search

BFS — алгоритм обхода графа, который исследует все вершины текущего уровня перед переходом к следующему. Идеально подходит для невзвешенных графов, где все рёбра имеют одинаковый вес.

Обход по уровням

Посещает все соседние вершины, затем переходит к их соседям

Очередь FIFO

Использует простую очередь для хранения вершин

Оптимален для невзвешенных графов

Гарантирует кратчайший путь по количеству рёбер

bfs.js 
function bfs(graph, start) {
  const queue = [start];
  const visited = new Set([start]);
  const distance = { [start]: 0 };
  
  while (queue.length > 0) {
    const node = queue.shift();
    
    for (const neighbor of graph[node]) {
      if (!visited.has(neighbor)) {
        visited.add(neighbor);
        distance[neighbor] = distance[node] + 1;
        queue.push(neighbor);
      }
    }
  }
  return distance;
}

Принцип работы BFS

1
Начало
Добавляем стартовую вершину в очередь
2
Извлечение
Берём вершину из начала очереди (FIFO)
3
Обход соседей
Посещаем всех непосещённых соседей
4
Повторение
Продолжаем, пока очередь не пуста

Принцип работы Дейкстры

1
Инициализация
Устанавливаем расстояние до старта = 0, остальные = ∞
2
Выбор минимума
Берём вершину с минимальным расстоянием из очереди
3
Релаксация
Обновляем расстояния до соседей, если нашли короче
4
Завершение
Повторяем, пока все вершины не обработаны
Алгоритм Дейкстры

Dijkstra's Algorithm

Алгоритм Дейкстры находит кратчайшие пути от стартовой вершины до всех остальных в взвешенных графах с неотрицательными весами рёбер. Использует жадный подход и приоритетную очередь.

Приоритетная очередь

Всегда выбирает вершину с минимальным текущим расстоянием

Релаксация рёбер

Обновляет кратчайшие расстояния при нахождении более короткого пути

Универсальность

Работает с любыми неотрицательными весами рёбер

dijkstra.js 
function dijkstra(graph, start) {
  const dist = {};
  const pq = new PriorityQueue();
  
  for (const node in graph) {
    dist[node] = Infinity;
  }
  dist[start] = 0;
  pq.enqueue(start, 0);
  
  while (!pq.isEmpty()) {
    const [node, d] = pq.dequeue();
    
    if (d > dist[node]) continue;
    
    for (const [neighbor, weight] of graph[node]) {
      const newDist = dist[node] + weight;
      if (newDist < dist[neighbor]) {
        dist[neighbor] = newDist;
        pq.enqueue(neighbor, newDist);
      }
    }
  }
  return dist;
}

Когда использовать BFS?

  • • Поиск друзей в социальных сетях (2-й, 3-й круг)
  • • Поиск кратчайшего пути в лабиринте без весов
  • • Проверка двудольности графа
  • • Поиск компонент связности

Когда использовать Дейкстру?

  • • Навигационные системы (Google Maps, GPS)
  • • Сетевые маршрутизаторы (протокол OSPF)
  • • Логистика и оптимизация маршрутов
  • • Игры с перемещением по взвешенной карте

BFS: Обход по уровням
BFS (Поиск в ширину): Алгоритм посещает все вершины текущего уровня перед переходом к следующему. В данном примере все рёбра имеют одинаковый вес (1), поэтому BFS найдёт оптимальный путь.
Алгоритм Дейкстры: На каждом шаге выбирает вершину с минимальным расстоянием от старта. Обратите внимание, как алгоритм учитывает веса рёбер и может пропустить близкую вершину, если до неё длинный путь.

Заключение

BFS и алгоритм Дейкстры — фундаментальные инструменты для поиска кратчайших путей в графах. Выбор между ними зависит от задачи: BFS идеален для невзвешенных графов благодаря линейной сложности, в то время как Дейкстра незаменим для взвешенных сценариев, таких как навигация или сетевые протоколы.

Понимание различий между этими алгоритмами поможет выбрать правильный инструмент для оптимизации производительности ваших приложений.
